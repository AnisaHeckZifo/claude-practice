# Data Dictionary — Dairy Manufacturing PoC

Four CSV files constitute the synthetic dataset. All are generated by scripts in
[src/data_gen/](src/data_gen/) using parameters from [configs/scenarios.yaml](configs/scenarios.yaml).

---

## 1. `data_raw/runs.csv`

One row per production run. This is the "header" table that links every time-series
row and lab result back to a specific run, product, scale, and scenario.

| Column | Type | Values / Range | Description |
|---|---|---|---|
| `run_id` | string | `RUN-0001` … | Unique run identifier |
| `product` | category | `QUARK`, `HIGH_PROTEIN_PUDDING` | Product manufactured |
| `scale` | category | `RD`, `TECHNIKUM`, `PRODUCTION` | Vessel / line scale |
| `scenario` | category | See scenarios_* in YAML | Fault scenario injected |
| `run_date` | date | ISO 8601 | Calendar date run started |
| `run_start_ts` | datetime | ISO 8601 | Timestamp of first sensor row |
| `run_end_ts` | datetime | ISO 8601 | Timestamp of last sensor row |
| `total_duration_hr` | float | Quark 12–15 h; Pudding 2–4 h | Actual elapsed run time |
| `batch_size_L` | float | RD 10–50; Technikum 100–500; Production 1000–10000 | Nominal batch volume |
| `separator_speed_setpoint_rpm` | float \| null | 5500–7500 rpm | **Quark only** — operator-entered centrifuge bowl speed setpoint; null for Pudding |
| `gel_break_time_min` | float \| null | 5–25 min | **Quark only** — actual duration of gel_break_mixing step; longer values widen variance of separation_deltaP, yield_loss_pct, and protein_pct; null for Pudding |
| `milk_tank_ids` | string | e.g. `"T17+T22"` | Both — IDs of milk/base-liquid tanks blended for this run |
| `blend_ratio` | string | e.g. `"0.6/0.4"` | Both — fractional contribution of each tank in milk_tank_ids order |
| `raw_protein_pct` | float \| null | 2.8–3.8 % | Both, optional — raw milk protein from blend; drives fermentation rate (Quark) and viscosity_proxy baseline (Pudding) |
| `raw_solids_pct` | float \| null | 11.0–13.5 % | Both, optional — raw milk total solids from blend |
| `quality_deviation` | int | 0, 1 | 1 = quality deviation recorded against this run |
| `yield_loss_pct` | float | 0–100 | % of planned yield lost |
| `downtime_minutes` | float | 0–480 | Unplanned downtime during run |
| `fouling_grade` | int | 0, 1, 2, 3 | 0=clean … 3=heavy; primarily Pudding, occasionally Quark separation |
| `extra_cleaning` | int | 0, 1 | 1 = unscheduled CIP triggered |
| `notes` | string | free text | Optional operator note (Faker-generated) |

**Key relationships:**
- `run_id` → `timeseries.csv` (many rows), `lab_results.csv` (one row), `events.csv` (many rows)
- `scenario` + `product` jointly determine which fault parameters are active

## 2. `data_raw/timeseries.csv`

One row per sample per run. Default cadence: **1 minute**. Quark produces
~720–900 rows per run; Pudding ~120–240 rows.

| Column | Type | Unit | Description |
|---|---|---|---|
| `run_id` | string | — | Foreign key → runs.csv |
| `t_min` | float | min | Elapsed minutes from run_start_ts |
| `timestamp` | datetime | ISO 8601 | Absolute timestamp |
| `step` | category | See process_steps in YAML | Active process step at this sample time |
| `temperature_C` | float | °C | Process temperature (step-dependent range; see YAML) |
| `pressure_bar` | float | bar | Line pressure |
| `flow_rate_lpm` | float | L/min | Volumetric flow rate |
| `shear_rpm` | float \| null | rpm | Agitator / pump RPM (0 during resting fermentation; null during separation — centrifuge uses `separator_speed_rpm` instead) |
| `separator_speed_rpm` | float \| null | rpm | **Quark, separation step only** — centrifuge bowl speed; ramps from 0 to setpoint over 3–8 min at step start; noise SD × 4 during deceleration at end-of-run; null outside separation step and for Pudding |
| `pH` | float | — | Inline or at-line pH (Quark: key fermentation signal; Pudding: near-constant ~6.65) |
| `conductivity_mS` | float \| null | mS/cm | **Quark only** (optional); null for Pudding and if not instrumented |
| `separation_deltaP` | float \| null | bar | **Quark only** — differential pressure across centrifuge; null outside separation step; variance scales with gel_break_time_min |
| `viscosity_proxy` | float \| null | mPa·s | **Pudding only** — inline viscometer proxy; null for Quark |
| `deltaT_heat_exchanger` | float \| null | °C | **Pudding only** — temperature difference across HX; primary fouling signal; null for Quark |
| `fouling_index` | float | 0–1 | Synthetic ground-truth fouling severity (0=clean); injected by scenario generator |
| `anomaly_flag` | int | 0, 1 | 1 = row falls within an injected fault window |

**Notes:**
- Null columns for the "wrong" product are included in the file as empty fields to keep
  a single unified schema; dashboards should filter on `product` before plotting.
- `fouling_index` and `anomaly_flag` are the labels that anomaly-detection models are
  trained to predict.

## 3. `data_raw/lab_results.csv`

One row per run — end-of-run QC / analytical results. Cadence is the run itself,
not a time-series. Join to `runs.csv` on `run_id`.

| Column | Type | Unit | Products | Description |
|---|---|---|---|---|
| `run_id` | string | — | both | Foreign key → runs.csv |
| `product` | category | — | both | `QUARK` or `HIGH_PROTEIN_PUDDING` |
| `sample_time` | datetime | ISO 8601 | both | When sample was drawn (typically end of filling step) |
| `protein_pct` | float | % | both | Protein content. Quark target 10–14 %; Pudding 8–15 % |
| `total_solids_pct` | float | % | both | Total solids. Quark 18–23 %; Pudding 20–28 % |
| `viscosity_value` | float | cP | both | Brookfield / rheometer viscosity. Quark 4000–15000 cP; Pudding 2000–8000 cP |
| `final_pH_offline` | float | — | both | Offline pH measurement. Quark: end-of-fermentation pH, target 4.4–4.7. Pudding: post-heat-treatment pH, target 6.3–7.0 |
| `d50_um` | float | µm | both | D50 particle size by laser diffraction. Quark: curd particles after centrifugation, normal ~28 µm; Pudding: starch granules + protein aggregates, normal ~14 µm. Elevated in SEPARATION_ISSUE (Quark) and FOUL (Pudding) |
| `fermentation_time_hr` | float \| null | h | Quark | Actual hours from inoculation to target pH; 99.0 = sentinel for STALL_FERMENT (run did not complete); null for Pudding |
| `result_flag` | category | `PASS`, `WARN`, `FAIL` | both | Overall QC verdict vs. spec_limits in YAML |

**Quark spec limits:** protein 10–14 %, total solids 18–23 %, viscosity 4000–15000 cP, final pH 4.4–4.7, fermentation time 10–16 h, D50 10–50 µm.
**Pudding spec limits:** protein 8–15 %, total solids 20–28 %, viscosity 2000–8000 cP, final pH 6.3–7.0, D50 5–25 µm.

## 4. `data_raw/events.csv`

One row per discrete event within a run. Events are instantaneous (or near-instantaneous)
markers that appear as vertical annotations on time-series plots.

| Column | Type | Values | Description |
|---|---|---|---|
| `run_id` | string | — | Foreign key → runs.csv |
| `event_id` | string | `EVT-00001` … | Unique event identifier |
| `timestamp` | datetime | ISO 8601 | When the event occurred |
| `t_min` | float | min | Elapsed minutes from run_start_ts |
| `step` | category | See process_steps in YAML | Active process step at event time |
| `event_type` | category | `inoculation`, `rennet_addition`, `gel_break_start`, `separation_start`, `standardization_start`, `filling_start`, `CIP`, `maintenance`, `changeover` | Event class |
| `product` | category | `QUARK`, `HIGH_PROTEIN_PUDDING` | Copied from run for convenience |
| `triggered_by` | category | `scheduled`, `fault`, `operator` | What caused the event |
| `duration_min` | float \| null | min | Duration if event has a window (CIP, maintenance); null for point events |
| `notes` | string | free text | Optional detail (Faker-generated for CIP/maintenance) |

**Event applicability by product:**

| Event | Quark | Pudding |
|---|---|---|
| `inoculation` | ✓ | — |
| `rennet_addition` | ✓ | — |
| `gel_break_start` | ✓ | — |
| `separation_start` | ✓ | — |
| `standardization_start` | ✓ | — |
| `filling_start` | ✓ | ✓ |
| `CIP` | ✓ | ✓ |
| `maintenance` | ✓ | ✓ |
| `changeover` | ✓ | ✓ |

## Entity-relationship summary

```
runs.csv (1)
  ├─── (N) timeseries.csv        [run_id, t_min, step, sensors...]
  ├─── (1) lab_results.csv       [run_id, QC attributes...]
  └─── (N) events.csv            [run_id, event_id, timestamp, event_type...]
```

All joins are on `run_id`.
